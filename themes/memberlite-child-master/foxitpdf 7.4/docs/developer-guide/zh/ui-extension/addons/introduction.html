# 插件化介绍

目录lib/uix-addons提供了丰富的、可自由组合的插件。内容如下：

    uix-addons
        ├─edit-graphics         ----- 编辑页面对象
        ├─text-object           ----- 编辑文本对象
        ├─export-form           ----- 导出表单
        ├─file-property         ----- 文档信息
        ├─full-screen           ----- 全屏功能
        ├─h-continuous          ----- 水平连续页面布局
        ├─h-facing              ----- 水平封面页面布局
        ├─h-single              ----- 水平单页页面布局
        ├─import-form           ----- 导入表单
        ├─multi-media           ----- 多媒体注释
        ├─password-protect      ----- 密码保护
        ├─path-objects          ----- 编辑path页面对象
        ├─print                 ----- 打印
        ├─redaction             ----- 密文功能
        ├─undo-redo             ----- undo-redo 功能
        └─allInOne.js           ----- 以上插件合集

每个addon对象都通过静态方法getName()(详见后文插件脚本入口)来标识，allInOne.js中addon的名称如下：

    'edit-graphics'
    'exportForm'
    'file-property'
    'full-screen'
    'h-continuous'
    'h-facing'
    'h-single'
    'importForm'
    'multi-media'
    'password-protect'
    'edit-pageobjects'
    'print'
    'redaction'
    'editTextObject'
    'undo-redo'

# allInOne.js

此文件为所有所有提供addon的合集。使用方法：

<script src="path/to/UIExtension.full.js"></script>
<script src="path/to/allInOne.js"></script>
<script>
    var pdfui = new UIExtension.PDFUI({...
    addons:UIXAddons,//UIXAddons is the library name in allInOne.js
    ...
    })
</script>

# 插件源码结构

# 入口文件(addon.info.json)

addon.info.json 文件为插件入口. 该文件指定了插件名称、多语言自持以及样式表. 内容如下：

{
    "library": "ExampleUIXAddon",
    "i18n": {
        "ns": "example",
        "sources": {
            "en-US": "./locales/en-US.json",
            "zh-CN": "./locales/zh-CN.json"
        }
    },
    "css": [
        "./index.css"
    ]
}

# 多语言"i18n"

该字段用于指定多语言. "ns"字段表示命名空间. "sources"字段指定各语言的资源文件.

配置后, 可使用 [i18n-namespace]:[i18n-key] 格式进行本地化.

此例中, i18n的命名空间为"example", 可使用的多语言为 "toolbar.title", "dialog.title" 或者 "buttons.addText"，详见zh-CN.json

如：

<h6>example:dialog.title</h6>

将被本地化成

<h6>Dialog title</h6>

以及

<h6>对话框标题</h6>

# "css" 字段

指定插件的样式表。需注意的是，指定的是插件样式表输出文件（仅css可用）。

# 插件脚本入口

一个插件类有三个静态函数：

static getName(){...} //Return addon's name.
static getLoader(){...} //Loader for addon.
static init(){...}  //Called after an addon is loaded. This would register some UI component.

以及一个成员函数：

fragments(){...} //Customize UI such as toolbar, side pannel.

请参阅源码实现

# 如何使用及自定义addon

如果用户仅使用部分addon，可按照如下方式使用：

<script src="path/to/UIExtension.full.js"></script>
<script src="path/to/allInOne.js"></script>
<script>
    UIXAddons = UIXAddons.filter(addon=>addon.getName()!= 'editTextObject')
    var pdfui = new UIExtension.PDFUI({...
    addons:UIXAddons,
    ...
    })
</script>

或者

<script src="path/to/UIExtension.full.js"></script>
<script>
    var pdfui = new UIExtension.PDFUI({...
    addons:[
        "path/to/customized-addon/addon.info.json",
        "path/to/lib/multi-media/addon.info.json",
        ...
    ],
    ...
    })
</script>
