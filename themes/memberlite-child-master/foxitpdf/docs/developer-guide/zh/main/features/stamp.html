# 自定义接口

ViewerCtrl 的 PDFViewer 提供了诺干接口以满足自定义需求：

接口 说明
PDFViewer.initAnnotationIcons(icons) 初始化标注图标(设置后，WEB SDK默认图标将不显示)
PDFViewer.addAnnotationIcon(icon) 添加单个图标
PDFViewer.removeAnnotationIcon(type,category,name) 移除单个图标
PDFUI.getAnnotationIcons(annotType,onlyCustomized) 获取自定义图标
StateHandlerManager.switchTo(name,params) 切换到 addStampStateHandler
PDFViewer.setFormatOfDynamicStamp(seperator,timeFormat) 设置动态信息格式
PDFPage.addAnnot(json) 指定存在的图标作为 stamp 的样式添加 stamp

# category 及 name 作用

UIExtension 提供的默认图标中，"Static", "SignHere", "Dynamic"等节点表示的是图标的 category，category 的子节点表示的是图标的 name，如：

  • "Approved"
  • "Confidential"
  • "Received"
  • "Reviewed"
  • "Revised"

**注意**:调用 PDFViewer.addAnnotationIcon(icon)添加自定义图标时，图标参数中也需指定 category 及 name。

category 及 name 的主要作用为用于指定的 stamp 的外观。

  • 在接口 PDFPage.addAnnot 中，当添加 stamp 时可通过 category 及 name 来指定 icon（需先调用PDFViewer.initAnnotationIcons(icons)初始化默认图标）：
pdfpage.addAnnot({
    type:PDFViewCtrl.PDF.annots.constant.Annot_Type.stamp,
    icon:'Approved',
    iconCategory:'Static'
    ...
})

# category 及 name 取值范围

接口 pdfui.getAnnotationIcons()用于获取已存在的图标，从中可以确定 category 及 name 取值范围。如何获取category 及 name 取值范围，样例如下：

let allIcons = pdfui.getAnnotationIcons(PDFViewCtrl.PDF.annots.constant.Annot_Type.stamp, false);
let iconNames = [];
for (let categoryKey in allIcons) {
	let category = allIcons[categoryKey];
	for (let name in category) {
		iconNames.push({
			category: categoryKey,
			name,
		});
	}
}

数组iconNames的每个元素即为category与name的合法值。

# 如何使用 icons

# 在 UIExtension 中使用

  • 终端用户添加 stamp 终端用户点击位于 UIExtension 的工具栏上的 stamp list 按钮，选择对应图标，即可在页面中添加预定外观的 stamp。 工具栏的 stamp list 底部，点击自定义 stamp 可添加自定义图标

  • 开发者设置默认状态为添加 stamp A - 通过 PDFUI 构造函数的 defaultStateHandler 及 handlerParams 属性设置，如下：

pdfui = new UIExtension.PDFUI({
	...
	customs:{
	defaultStateHandler: PDFViewCtrl.STATE_HANDLER_NAMES.STATE_HANDLER_CREATE_STAMP
	handlerParams:{
		category:'SignHere',
		name:'SignHere'
	}
	}
	...
})

B - 通过StateHandlerManager.switchTo()实现，如：

pdfui.getStateHandlerManager().then((handlerManager) =>
	handlerManager.switchTo(PDFViewCtrl.STATE_HANDLER_NAMES.STATE_HANDLER_CREATE_STAMP, {
		category: 'SignHere',
		name: 'SignHere',
	})
);

# 在 PDFViewerCtrl 中使用

使用 PDFPage.addAnnot 接口添加 stamp，可指定使用预定义或自定义的图标。

# 使用预定义 icon

在使用预定义icon前，需先调用PDFViewer.initAnnotationIcons(icons)设置所有预定义的icon。其后，PDFPage.addAnnot参数中指定预定义icon的category及name，如下：

pdfPage.addAnnot({
	...
	icon:"name",//icon name
	iconCategory:"category",//icon category
	...
})

需要注意的是，PDFPage.addAnnot指定的预定义icon必须在PDFViewer.initAnnotationIcons(icons)中，否则添加的stamp无法显示外观(可通过PDFViewer.addAnnotationIcon(add)接口增加预定义icon)。

# 使用非预定义 icon

通过指定 PDFPage.addAnnot 参数中的 iconInfo 属性来使用非预定义 icon, 如下：

pdfPage.addAnnot({
	...
	iconInfo:{
		category:"category",
		name:"name",
		fileType:"bmp",
		url:"http://XX.XX"
	}
	...
})

当添加成功后，icon 将自动变成预定义 icon（不会同步到 UIExtension，如果需要同步，请使用 PDFViewer.addAnnotationIcon(icon)接口），设置后添加stamp可参照前文使用。
