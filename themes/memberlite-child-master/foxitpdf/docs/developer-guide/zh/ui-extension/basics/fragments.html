# UI fragments

fragment是一段配置，定义了如何修改以及删除、插入等UI操作，使用它的目的是方便基于内置模板进行少量的UI定制。 如果需要大量的自定义布局和设备自适应， 请参考 Appearance 和 layout template 中描述的方法

# simple example

下面是利用fragment配置分别删除mobile和desktop/tablet布局中的comment-tab组件, 请点击“run”运行示例，mobile/tablet的效果可以使用 Chrome DevTool 的 device mode 来模拟（simulate）运行

<script>
    var CustomAppearance= UIExtension.appearances.adaptive.extend({
        getDefaultFragments: function() {
            var isMobile = PDFViewCtrl.DeviceInfo.isMobile;
            if(isMobile) {
                // 针对mobile设备的fragment配置
                return [{
                    target: 'comment-tab',
                    action: 'remove'
                }];
            } else {
                // 针对桌面以及平板设备的fragment配置
                return [{
                    target: 'comment-tab',
                    action: 'remove'
                }, {
                    target: 'hand-tool',
                    config: {
                        callback: {
                            around: function(callback, args) {
                                try{
                                    console.info('before callback');
                                    var ret;
                                    if(callback instanceof UIExtension.Controller) {
                                        ret = callback.handle(...args);
                                    } else {
                                        ret = callback.apply(this, args);
                                    }
                                    console.info('after callback');
                                    return ret;
                                }catch(e) {
                                    console.error(e, 'an error occurred');
                                } finally {
                                    console.info('');
                                }
                            }
                        }
                    },
                }];
            }
        }
    });
    var libPath = window.top.location.origin + '/lib';

    var pdfui = new UIExtension.PDFUI({
            viewerOptions: {
                libPath: libPath,
                jr: {
                    licenseSN: licenseSN,
                    licenseKey: licenseKey
                }
            },
            renderTo: document.body,
            // 根据设备类型提供不同的 appearance
            appearance: CustomAppearance,
            addons: []
    });
</script>

{
    "iframeOptions": {
        "style": "height: 500px",
        "injectCss": [
            "/lib/UIExtension.vw.css"
        ]
    }
}

# fragment配置参数描述

  • target 控件的名称， 每个控件的名称都是唯一的
  • action 表示这个片段的作用方式，默认为 UIExtension.UIConsts.FRAGMENT_ACTION.EXT, 具体说明如下：
    • UIExtension.UIConsts.FRAGMENT_ACTION.EXT 表示扩展目标控件的配置
    • UIExtension.UIConsts.FRAGMENT_ACTION.BEFORE 表示在目标控件前插入控件
    • UIExtension.UIConsts.FRAGMENT_ACTION.AFTER 表示在目标控件后插入控件
    • UIExtension.UIConsts.FRAGMENT_ACTION.APPEND 表示将控件插入到目标控件，要求目标控件必须是容器类型的，否则报错
    • UIExtension.UIConsts.FRAGMENT_ACTION.FILL 表示将目标控件的子空间全部情况并用新控件覆盖，要求目标控件必须是容器类型，否则报错
    • UIExtension.UIConsts.FRAGMENT_ACTION.REPLACE 表示将目标控件替换为新控件
    • UIExtension.UIConsts.FRAGMENT_ACTION.REMOVE 表示删除目标控件
  • template 控件模版， 内容为XML格式，action为 BEFORE/AFTER/APPEND/FILL/REPLACE 才有效
  • config 控件配置对象，action为 REMOVE 时无效
    • config.target 上面的template模板中控件的名称，只有 action 为 BEFORE/AFTER/APPEND/FILL/REPLACE 时需要用到
    • config.attrs 设置控件的html属性
    • config.callback 控件的业务逻辑实现， 有以下三种写法：

      • function， 控件的事件会调用到这个函数，而且会覆盖内置回调。支持function的基础组件只有（xbutton,dropdown-button,context-menu-item）。如果要基于内置回调添加功能，可以用第二种方法。

      • controller class: Controller类可以监听组件生命周期以及处理更多的组件事件：

        {
    target: 'hand-tool',
    config: {
        callback: class extends UIExtension.Controller {
            mounted() {
                super.mounted();
                this.component.element.addEventListener('hover', e => {
                    console.info('mouse over', this.component)
                })
            }
            handle() {
                console.info('hand-tool clicked')
            }
        }
    }
}

      • decorator object: 代理对象，可以在原始回调函数 之前/之后/异常 插入代码：

        {
    target: 'hand-tool',
    config: {
        callback: {
            before: function() {
                // before button click callback
            },
            after: function(returnValue) {
                // after button click callback called
            },
            thrown: function(error) {
                // button click callback throws an error
            },
            around: function(callback, args) {
                try{
                    console.info('before callback');
                    var ret;
                    if(callback instanceof UIExtension.Controller) {
                        ret = callback.handle(...args);
                    } else {
                        ret = callback.apply(this, args);
                    }
                    console.info('after callback');
                    return ret;
                }catch(e) {
                    console.error(e, 'an error occurred');
                } finally {
                    console.info('');
                }
            }
        }
    }
}

# 注意事项

建议 fragment 只用于小部分UI调整时使用，如果对内置布局修改较大，建议参考 Appearance 和 layout template 中描述的方法
